01

배경 및 문제 정의

현재 재무비교분석기는 Python 스크립트 형태의 PoC로 존재한다. 사용자가 회사명을 코드에 직접 지정하면, DART Open API에서 corp_code를 조회하고 최신 사업보고서의 3개년 연결재무제표와 배당 공시를 가져와 엑셀 파일(원본데이터·비교분석·대시보드 3개 시트)로 산출한다. 지금까지 삼성전자·SK하이닉스·현대자동차, 그리고 한국전력·LG에너지솔루션·HD현대 조합으로 두 차례 수동 실행되었다.

이 방식의 한계는 다음과 같다.

  • 비교 대상 기업명을 바꿀 때마다 코드를 수정하고 재실행해야 하며, 결과 확인까지 개발자 개입이 필요하다.
  • 비교 기업 수(3개)와 연도 수(3개년)가 스크립트에 고정되어 있어 유연한 비교가 불가능하다.
  • 산출물이 매번 새 엑셀 파일로만 남아, 과거 비교 결과를 검색하거나 다시 열람하기 어렵다.
  • 계정과목명이 기업마다 달라(예: "당기순이익" vs "연결당기순이익") 매칭 실패 가능성이 상존하지만, 이를 확인할 수 있는 화면이 없다.
기회. 검색·비교·저장의 반복 작업을 웹 UI로 옮기면, 별도 코딩 지식 없이 개인 투자자나 주니어 애널리스트가 스스로 원하는 기업 조합을 비교할 수 있다.
02

목표 및 성공지표

제품 목표. 국내 상장기업의 재무제표를 회사명 검색만으로 비교하고, 결과를 신뢰할 수 있는 근거(출처·수식)와 함께 엑셀로 저장할 수 있는 셀프서비스 도구를 만든다.

지표정의MVP 목표치
검색→비교 소요시간회사명 입력부터 비교표 렌더링까지10초 이내
동시 비교 기업 수한 번의 비교 세션에 포함 가능한 기업 수최대 5개
비교 가능 연도 수사업보고서 기준 조회 가능 연도 범위최대 5개년
엑셀 내보내기 성공률수식 오류 없이 다운로드 완료되는 비율100%
계정 매칭 실패 표기율매칭 실패 시 "N/A" 및 사유가 노출되는 비율100%
03

대상 사용자

  • 개인 투자자 — 관심 종목과 동종업계 경쟁사의 수익성·안정성을 빠르게 대조하고 싶은 사용자.
  • 회계·재무 전공 학생 및 주니어 애널리스트 — 재무비율 산출 근거를 학습하거나 리포트 초안 자료로 활용하려는 사용자.
  • 중소기업 재무 담당자 — 동종업계 상장사와 자사 재무구조를 참고 비교하려는 사용자.
04

범위 정의

1차 출시(MVP) 기준으로 아래와 같이 범위를 나눈다.

In scope

  • 국내 상장사 한글 회사명 검색
  • 2~5개 기업 동시 비교
  • 최근 최대 5개 사업연도 비교
  • 수익성·안정성·성장성·배당 지표
  • 추이 차트 및 요약 대시보드
  • 수식 기반 엑셀 내보내기
  • 데이터 출처(공시/연도/연결·별도) 표기

Out of scope (1차)

  • 해외 상장사 비교
  • 실시간 시세·주가 연동
  • 투자자문 또는 매수·매도 추천
  • 모바일 네이티브 앱
  • 회원가입/로그인, 비교 이력 클라우드 저장
  • 업종 평균 벤치마크 (2차 검토)
05

기능 요구사항

P0 필수 — MVP 출시 조건 P1 우선 — 출시 직후 빠르게 추가 P2 검토 — 수요 확인 후 진행

5.1회사 검색

우선순위요구사항
P0한글 회사명 입력 시 DART corpCode 캐시 기반 자동완성을 제공한다.
P0동일 회사명이 상장/비상장으로 중복될 경우 종목코드를 함께 노출해 구분한다 (예: 현재 스크립트의 find_corp_code 로직 — 상장사 우선 선택).
P1최근 검색·비교 조합을 브라우저 로컬에 저장해 재사용을 돕는다.

5.2비교 설정

우선순위요구사항
P02~5개 기업을 동시에 선택해 비교 세션을 구성할 수 있다.
P0최근 사업보고서를 기준으로 최대 5개년(현재는 3개년 고정) 데이터를 선택할 수 있다.
P1연결(CFS)/별도(OFS) 재무제표를 사용자가 직접 전환할 수 있다 (현재는 CFS→OFS 자동 폴백만 존재).

5.3재무비율 엔진

우선순위지표군세부 항목
P0수익성매출총이익률 · 영업이익률 · 순이익률 · ROA · ROE
P0안정성부채비율 · 유동비율 · 자기자본비율
P0성장성전년대비 매출액 · 영업이익 · 순이익 · 총자산 증가율
P0배당배당성향 (현금배당금총액 ÷ 당기순이익)
P1배당·활동성 확장배당수익률(주가 연동 필요), 당좌비율, 총자산회전율
P2벤치마크업종 평균 대비 상대 위치 표시

5.4시각화

우선순위요구사항
P0매출액·영업이익·당기순이익 추이를 기업별 라인차트로 제공한다.
P0ROE·부채비율·배당성향 등 주요 비율을 기업별 막대 비교차트로 제공한다.
P1최신연도 요약 스코어카드에 조건부 서식(색상 스케일)을 적용해 한눈에 비교하게 한다.
P2다차원 지표를 레이더 차트로 동시에 비교하는 옵션을 제공한다.

5.5내보내기 및 공유

우선순위요구사항
P0현재 구조(원본데이터·비교분석·대시보드 시트, 수식 기반 계산)를 유지한 엑셀 다운로드를 제공한다.
P1PDF 요약 리포트를 다운로드할 수 있다.
P2읽기 전용 공유 링크로 비교 결과를 외부에 전달할 수 있다.
06

비기능 요구사항

  • 데이터 최신성 — DART 공시 갱신에 맞춰 최소 1일 1회 재조회·캐시 갱신을 수행한다.
  • API 레이트리밋 대응 — DART Open API의 개인키 일일 호출 한도를 고려해 corp_code 목록과 재무데이터를 서버 사이드에 캐싱하고, 동일 기업·연도 재조회를 최소화한다.
  • 정확성과 투명성 — 계정과목 매칭 실패 시 하드 에러 대신 "N/A"와 실패 사유를 노출한다 (현재 pick_item 로직의 alias·substring 매칭 원칙 유지).
  • 성능 — 캐시 히트 기준 비교 결과 렌더링 3초 이내.
  • 보안 — DART API 키는 서버에만 보관하며 클라이언트로 노출하지 않는다.

10

리스크 및 미결정 사항

구분내용
데이터 매칭계정과목명이 기업·업종별로 상이해(예: "당기순이익" vs "연결당기순이익") alias·substring 매칭이 완벽하지 않을 수 있음.
배당 데이터 공백alotMatter 응답의 전전기(bfefrmtrm) 필드가 일부 기업·연도에서 누락되는 사례 확인(HD현대 2023). 원인 조사 필요.
API 호출 한도무료 개인키의 일일 호출 한도 초과 시 대응 정책(대기열, 캐시 우선 제공 등) 미정.
과금·인증 모델개인용 무료 도구로 유지할지, 로그인·유료 티어를 둘지 미정.
